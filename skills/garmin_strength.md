# RunLoop Strength & Mobility – Kräftigung, Mobilität und Beschwerdemanagement

Dieser Skill erstellt individuelle Kräftigungs- und Mobilitätsroutinen für Läufer, reagiert auf körperliche Beschwerden und integriert alles in den Trainingsplan.

## Wann wird dieser Skill genutzt?

1. **Routine-Erstellung**: Nutzer fragt nach Kräftigungs- oder Mobilitätsübungen
2. **Beschwerden**: Nutzer meldet Schmerzen, Verspannungen oder Einschränkungen
3. **Prävention**: Als Teil der Trainingsplanung (z.B. an Ruhetagen)
4. **Nach Analyse**: Wenn Running Dynamics oder Verletzungsrisiko-Indikatoren auffällig sind

## Beschwerde-Reaktion

Wenn der Nutzer über Beschwerden klagt, folge diesem Ablauf:

### Schritt 1: Einordnung

| Beschwerde | Mögliche Ursache | Dringlichkeit |
|-----------|-----------------|---------------|
| Knieschmerzen (vorne, unter Kniescheibe) | Runner's Knee / Patellasehne | Mittel |
| Knieschmerzen (außen) | IT-Band Syndrom (ITBS) | Mittel |
| Schienbeinschmerzen | Shin Splints / Periostitis | Mittel |
| Achillessehne (morgens steif) | Achillodynie (Überlastung) | Hoch |
| Plantarfaszie (Ferse morgens) | Plantarfasziitis | Hoch |
| Hüfte/Leiste | Hüftbeuger-Überlastung oder Adduktoren | Mittel |
| Unterer Rücken (nach langem Lauf) | Core-Schwäche, Hüftbeuger-Verkürzung | Niedrig |
| Wade (Verhärtung, Zerrung) | Muskuläre Überlastung | Mittel |
| Fußrücken | Stressfraktur-Risiko | Hoch – Arzt! |

### Schritt 2: Sofortmaßnahme

- **Hoch (Achilles, Plantarfaszie, Fußrücken)**: Laufpause empfehlen, Arztbesuch vorschlagen
- **Mittel**: Volumen reduzieren, spezifische Übungen erstellen
- **Niedrig**: Übungen integrieren, Training fortsetzen

### Schritt 3: Individuelles Programm erstellen

Erstelle ein Programm mit:
- 4-6 Übungen, laufspezifisch auf die Beschwerde zugeschnitten
- Frequenz (täglich, jeden 2. Tag, vor/nach dem Lauf)
- Dauer (10-20 min)
- Progression (Woche 1 → Woche 4)

### Schritt 4: In Plan integrieren

Speichere das Programm in `data/strength_plan.json` und zeige es im Dashboard.

## Routine-Typen

### Pre-Run Aktivierung (5-8 min)
Dynamische Übungen VOR dem Lauf:
- Leg Swings (10 pro Seite)
- Walking Lunges (8 pro Seite)
- High Knees (20 Sek)
- A-Skip / B-Skip (10 pro Seite)
- Hüftkreisen (5 pro Richtung)
- Squat to Stand (5x)

### Post-Run Dehnung (10 min)
Statische Übungen NACH dem Lauf:
- Wadendehnung (60 Sek pro Seite)
- Quadrizeps-Dehnung (60 Sek pro Seite)
- Hüftbeuger-Stretch (60 Sek pro Seite)
- Pigeon Stretch (60 Sek pro Seite)
- Hamstring-Stretch (60 Sek pro Seite)

### Ruhetag-Routine: Kraft (20-25 min)
An trainingsfreien Tagen:
- Auswahl aus Kräftigungskatalog basierend auf Schwächen
- Fokus auf Hüftstabilität und Core
- 3 Runden, kurze Pausen (30-60 Sek)

### Ruhetag-Routine: Mobilität (15 min)
An trainingsfreien Tagen:
- Auswahl aus Mobilitätskatalog
- Fokus auf Hüfte und Sprunggelenk
- Langsam, kontrolliert, Atmung beachten

## Integration in den Trainingsplan

Im `data/plan.json` werden Kraft/Mobility-Einheiten als eigene Tage oder als Zusatz zu Ruhetagen eingeplant:

```json
{
  "datum": "2026-06-05",
  "typ": "kraft_mobility",
  "beschreibung": "Kraft-Routine: Hueftstabilitaet + Core",
  "dauer_min": 20,
  "details": "Einbeinige Bruecke 3x12, Clamshells 3x15, Seitlicher Plank 3x30s, Dead Bug 3x10",
  "coach_notiz": "An Ruhetagen: Kraft baut die Stabilitaet auf die dein Laufen sicherer macht."
}
```

## Dashboard-Integration

Im Training-Tab werden Kraft/Mobility-Einheiten mit eigenem Badge angezeigt:
- Badge: "Kraft" (lila) oder "Mobility" (cyan)
- Übungen aufgelistet mit Wiederholungen
- Bei Beschwerden: Beschwerde-Programm hervorgehoben mit Fortschritts-Tracker

## Workflow

1. Nutzer meldet Beschwerde ODER fragt nach Übungen ODER Coach erkennt Bedarf
2. Passende Übungen aus dem Katalog wählen
3. Programm in `data/strength_plan.json` speichern
4. In den Trainingsplan (data/plan.json) integrieren
5. Dashboard aktualisieren
6. Fortschritt tracken (hat der Nutzer die Routine gemacht?)


## Übungsbibliothek (data/exercises_library.json)

JEDE Übung die jemals vorgeschlagen wird, MUSS in `data/exercises_library.json` gespeichert werden. Das ist die zentrale Referenz im Dashboard (Tab "Übungen").

### Pflicht-Felder pro Übung:

```json
{
  "id": "kebab-case-name",
  "name": "Anzeigename",
  "kategorie": "Pre-Run | Post-Run | Kraft | Mobilitaet | Rehab",
  "zielbereich": "Welche Muskeln/Gelenke",
  "level": "Anfaenger | Mittel | Fortgeschritten",
  "wiederholungen": "3x12 pro Seite / 60 Sek / etc.",
  "anleitung": "Detaillierte Schritt-fuer-Schritt-Anleitung. So ausfuehrlich wie moeglich.",
  "worauf_achten": ["Punkt 1", "Punkt 2", "..."],
  "haeufige_fehler": ["Fehler 1", "Fehler 2"],
  "varianten": ["Einfachere Version", "Schwierigere Version"]
}
```

### Regeln:
- JEDE neue Übung wird zur Bibliothek hinzugefügt
- Bestehende Übungen werden NICHT überschrieben
- Anleitungen so detailliert wie möglich (ein Anfänger muss sie verstehen)
- "worauf_achten" enthält die wichtigsten Ausführungshinweise
- "haeufige_fehler" warnt vor typischen Problemen
- "varianten" bietet Alternativen (einfacher/schwieriger)

### Dashboard-Feature:
- Tab "Übungen" zeigt ALLE Übungen aus der Bibliothek
- Live-Textfilter (Suche nach Name, Zielbereich, Kategorie)
- Kategorie-Buttons zum Filtern (Pre-Run, Post-Run, Kraft, Mobilität, Rehab)
- Sofortige Filterung bei Eingabe (kein Button-Klick nötig)
